# 評価軸カタログ

目的に応じて評価軸を選び、重みを設定する。全項目を使わず、案件に必要なものだけを採用する。

## コスト

| 評価軸 | 観点例 |
|---|---|
| 初期導入コスト | 導入費、移行費、トレーニング費 |
| 年間運用コスト | ライセンス、保守、人件費 |
| 変動コスト | 利用量増減時の費用影響 |

## 機能・品質

| 評価軸 | 観点例 |
|---|---|
| 必須要件適合度 | Must要件の充足率 |
| ユーザビリティ | 学習容易性、操作性 |
| 性能 | 応答時間、同時接続、処理能力 |
| 可用性 | SLA、障害復旧、冗長性 |

## 技術適合性

| 評価軸 | 観点例 |
|---|---|
| 既存システム連携 | API、データ連携、認証連携 |
| 拡張性 | 将来機能追加への対応 |
| 保守性 | 運用しやすさ、変更容易性 |
| ベンダーロックイン耐性 | 代替可能性、移行難易度 |

## セキュリティ・ガバナンス

| 評価軸 | 観点例 |
|---|---|
| セキュリティ統制 | 認証認可、監査ログ、脆弱性対策 |
| 法令・規制対応 | 個人情報保護、業界規制、監査要件 |
| データ管理 | 保管場所、暗号化、バックアップ |

## 導入実行性

| 評価軸 | 観点例 |
|---|---|
| 導入スピード | 契約から本番までの期間 |
| 移行難易度 | 既存データ移行、並行稼働の負荷 |
| 体制適合性 | 社内スキル、運用体制との適合 |
| ベンダー支援品質 | サポート体制、問い合わせ応答品質 |

## 事業インパクト

| 評価軸 | 観点例 |
|---|---|
| ROI見込み | 投資回収期間、費用対効果 |
| 業務改善効果 | 工数削減、品質向上、リードタイム短縮 |
| 戦略整合性 | 中期計画、部門方針との一致 |

## 重み設定の目安

- コスト重視: コスト系の合計重みを `35-50%` にする。
- 速度重視: 導入実行性と機能適合度の重みを高める。
- 安定運用重視: 可用性、保守性、統制系の重みを高める。
- 迷う場合: まず「目的への寄与が最も大きい3軸」を合計 `60%` 以上にする。
